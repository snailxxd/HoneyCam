# HoneyCam 项目阶段总结与实验指南（截至当前版本）

## 1. 已完成工作

### 1.1 系统原型能力

- 已完成 Spring Boot + Thymeleaf + Three.js 的蜜罐原型框架。
- 已实现伪装登录页，记录攻击者输入的用户名、密码、来源 IP、User-Agent。
- 已实现摄像头监控页与 360 全景渲染。
- 已实现 RTSP 554 握手仿真（`OPTIONS / DESCRIBE / SETUP / PLAY / PAUSE / TEARDOWN`）。

### 1.2 交互与日志

- 已实现交互日志记录（`/api/interaction`）。
- 已实现会话起止日志（`SESSION_START / SESSION_END`），可计算停留时长（Dwell Time）。
- 日志按 JSONL 持久化输出（登录、交互、RTSP 请求）。

### 1.3 分组能力（A/B/C 中的 B、C）

- 已实现 `group-b`（低交互组）：
  - 禁用用户 PTZ 输入；
  - 启用自动巡航（画面自动来回转动），模拟循环播放效果。
- 已实现 `group-c`（高交互组）：
  - 允许用户 PTZ 操作；
  - 支持全景媒体加载（图片默认，视频可选）。

### 1.4 欺骗细节增强

- 已加入预览启动延时与“Connecting to Camera...”进度条，模拟真实设备连接过程。
- 已支持全景媒体回退策略：视频失败自动降级到图片，避免黑屏。
- 已支持抓包脚本（Windows/Linux）和日志分析脚本（转化率、CDF 基础数据输出）。

## 2. 当前代码中的启动方式

### 2.1 本地开发运行（Windows PowerShell）

```powershell
.\gradlew.bat bootRun
```

访问：`http://localhost:8081/login`

### 2.2 Group B（低交互）

```powershell
.\gradlew.bat bootRun "--args=--spring.profiles.active=group-b"
```

### 2.3 Group C（高交互）

```powershell
.\gradlew.bat bootRun "--args=--spring.profiles.active=group-c"
```

## 4. 建议实验执行流程（可直接照做）

1. **准备环境**
   - 选择云服务器（建议 Ubuntu）；
   - 配置 Java 运行环境与项目部署；
   - 开放端口（HTTP 80、RTSP 554）。

2. **部署分组**
   - 部署 Group B（低交互自动巡航）；
   - 部署 Group C（高交互 PTZ）；
   - 准备 Group A（真实设备/替代数据）。

3. **启动抓包与日志采集**
   - 启动应用；
   - 启动 `tcpdump/tshark` 抓包；
   - 连续运行 7~10 天。

4. **实验期间巡检**
   - 每日检查日志文件是否持续增长；
   - 检查服务可用性（HTTP/RTSP）；
   - 记录异常（宕机、网络变更、端口阻断）。

5. **实验结束后分析**
   - 运行 `scripts/analyze_logs.py`；
   - 汇总三组指标并绘图；
   - 使用 Nmap/Shodan 做主动探测结果对比。

6. **撰写结论**
   - 回答核心问题：低成本 360 方案能否接近真实设备欺骗效果；
   - 明确说明边界条件（样本量、部署时长、网络环境偏差）。

---

## 5. 风险与注意事项

- 若使用外网资源（如在线全景图），可能受网络条件影响导致加载失败，建议实验环境改为本地静态资源。
- B/C 组应尽量保持除交互能力外的其他条件一致，避免对比偏差。
- 公网实验可能采集到敏感攻击数据，需做好日志权限隔离与合规管理。
